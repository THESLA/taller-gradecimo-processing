1 mousePressed() — Reaccionar al clic

Ejercicio 1: "Dibuja con clics"

mousePressed eventos

¿Qué vamos a aprender? Processing tiene funciones especiales que se ejecutan automáticamente cuando ocurre algo. mousePressed() se ejecuta cada vez que haces clic con el mouse. ¡Es como un draw() que solo corre cuando cliqueas!

Paso 1: Copia esto en tu cuaderno

// mousePressed: dibuja un círculo donde cliqueas

void setup() {
  size(400, 400);
  background(30, 30, 60);
}

void draw() {
  // Vacío: no necesitamos dibujar continuamente
}

void mousePressed() {
  fill(231, 76, 60);
  ellipse(mouseX, mouseY, 50, 50);
}

Paso 2: Escríbelo en Processing y ejecútalo

Abre Processing, escribe el código, haz clic en Play (▶) y después haz clic en cualquier parte del canvas. Cada clic dibuja un círculo rojo.

Modifica y experimenta

Cambia el color del fill(), prueba con diferentes valores RGB. ¿Qué pasa si cambias el tamaño del ellipse() a 80? ¿Y si pones background(30, 30, 60); dentro de mousePressed() en lugar de setup()?

Para pensar

¿Por qué el canvas no se borra cada vez que haces clic? ¿Qué pasaría si movemos background() a draw()?

2 random() — Sorpresa y variedad

Ejercicio 2: "Explosión de colores"

random mousePressed

¿Qué vamos a aprender? random() genera un número al azar entre 0 y el valor que le pases. Si escribes random(255), te da cualquier número entre 0 y 255. ¡Cada clic será una sorpresa!

Paso 1: Copia esto en tu cuaderno

// random: colores y tamaños aleatorios

void setup() {
  size(400, 400);
  background(30, 30, 60);
}

void draw() {
}

void mousePressed() {
  // fill con 3 valores aleatorios (R, G, B)
  fill(random(255), random(255), random(255));
  // ellipse con tamaño aleatorio entre 20 y 80
  ellipse(mouseX, mouseY, random(20, 80), random(20, 80));
}

Paso 2: Escríbelo en Processing y ejecútalo

Copia el código, ejecútalo y haz clic varias veces. Cada clic da un color y tamaño diferente. ¡Nunca dos clics son iguales!

Modifica y experimenta

1. Cambia random(20, 80) por random(10, 150).
2. Agrega random(100, 255) como cuarto valor del fill() para cambiar la transparencia (alpha).
3. Haz que background() también use random cada cierto tiempo.

Para pensar

¿Qué diferencia hay entre random(255) y random(20, 80)? ¿Cómo harías para que el color siempre tenga rojo alto (cerca de 255)?

3 Animación — Movimiento con variables

Ejercicio 3: "La pelota viajera"

variables animación draw

¿Qué vamos a aprender? draw() se ejecuta 60 veces por segundo. Si tenemos una variable x que empieza en 0 y en cada draw() hacemos x = x + 2, la figura se irá moviendo sola. ¡Esto es animación!

Paso 1: Copia esto en tu cuaderno

// Animación: una pelota que se mueve sola
float x = 0;

void setup() {
  size(400, 200);
}

void draw() {
  background(30, 30, 60);
  
  // Dibuja la pelota en la posición x
  fill(46, 204, 113);
  ellipse(x, 100, 40, 40);
  
  // Mueve la pelota: x aumenta cada frame
  x = x + 2;
  
  // Si la pelota sale por la derecha, vuelve a empezar
  if (x > width + 20) {
    x = -20;
  }
}

Paso 2: Escríbelo en Processing y ejecútalo

La pelota verde cruza la pantalla de izquierda a derecha una y otra vez. Prueba cambiando x = x + 2 por x = x + 5 o x = x + 0.5.

Modifica y experimenta

1. Agrega una variable float y = 100; y haz que la pelota también se mueva verticalmente con y = y + 1.5.
2. Combínalo con mousePressed() para que al hacer clic la pelota cambie de color con fill(random(255), random(255), random(255)).
3. ¿Puedes hacer que la velocidad también sea una variable?

4 keyPressed() — Control con teclado

Ejercicio 4: "Controla con las teclas"

keyPressed if variables

¿Qué vamos a aprender? Así como mousePressed() detecta clics, keyPressed() detecta cuando presionas una tecla. Podemos usar la variable key para saber qué tecla se presionó y hacer cosas diferentes con if.

Paso 1: Copia esto en tu cuaderno

// keyPressed: controla la pelota con teclas
float x = 200;
float y = 200;

void setup() {
  size(400, 400);
}

void draw() {
  background(30, 30, 60);
  fill(231, 76, 60);
  ellipse(x, y, 50, 50);
}

void keyPressed() {
  // La variable 'key' guarda la tecla presionada
  if (key == 'a') {
    x = x - 10;
  }
  if (key == 'd') {
    x = x + 10;
  }
  if (key == 'w') {
    y = y - 10;
  }
  if (key == 's') {
    y = y + 10;
  }
}

Paso 2: Escríbelo en Processing y ejecútalo

Presiona las teclas A, D, W, S para mover el círculo rojo. Observa cómo key guarda el carácter de la tecla que presionaste.

Modifica y experimenta

1. Agrega más teclas: por ejemplo, la barra espaciadora (se representa como ' ') para que la pelota vuelva al centro.
2. Combínalo con keyPressed() y random() para que al presionar 'c' la pelota cambie a un color aleatorio.
3. ¿Qué pasa si cambias key == 'a' por key == 'a' || key == 'A'?

Para pensar

¿Cómo harías para que la pelota no se salga del canvas? Pista: necesitas if para limitar x e y entre 0 y width/height.

5 Extra: La pelota que rebota (si te alcanza el tiempo)

Ejercicio 5: Rebote en los bordes

if velocidad desafío

Combinando animación e if, podemos hacer que la pelota rebote como en un videojuego. Cuando la pelota llega al borde, invertimos su velocidad.

Copia en tu cuaderno (si te alcanza el tiempo)

// Pelota que rebota en los 4 bordes
float x = 200;
float y = 200;
float velX = 3;
float velY = 2;

void setup() {
  size(400, 400);
}

void draw() {
  background(30, 30, 60);
  
  // Dibuja la pelota
  fill(46, 204, 113);
  ellipse(x, y, 40, 40);
  
  // Mueve la pelota
  x = x + velX;
  y = y + velY;
  
  // Rebota en los bordes
  if (x > width - 20 || x < 20) {
    velX = -velX;  // Invierte dirección horizontal
  }
  if (y > height - 20 || y < 20) {
    velY = -velY;  // Invierte dirección vertical
  }
}
Para pensar

¿Qué hace velX = -velX? ¿Porqué usamos width - 20 y no simplemente width? (Pista: el radio de la pelota es 20)

Modifica

Agrega mousePressed() para que al hacer clic la pelota se teletransporte a la posición del mouse. Agrega keyPressed() para que al presionar 'r' reinicie la posición.

6 Tu proyecto personal

Crea tu propia obra de arte interactiva

Usando todo lo que aprendiste (hoy y la clase anterior), crea un programa original. El tema es libre, pero debe incluir:

2+ figuras

Usa al menos dos figuras diferentes (ellipse, rect, line, triangle...)

Colores

fill(), stroke(), background() — usa colores que te gusten

mouseX / mouseY

Al menos una figura reacciona a la posición del mouse

Un if

Una condición que cambie algo (color, posición, tamaño...)

mousePressed()

Algo debe pasar cuando haces clic

random()

Usa random en al menos una parte (color, posición, tamaño)

keyPressed()

Al presionar una tecla debe ocurrir algo

Animación

Algo que se mueva solo (variables que cambian en draw)

Ideas para inspirarte:

• Un paisaje nocturno con estrellas que parpadean (random) y una luna que sigue al mouse
• Una cara que cambia de expresión al hacer clic (if + mousePressed)
• Figuras geométricas que orbitan alrededor del mouse
• Un "pincel mágico" que dibuja patrones de colores aleatorios
• Una pelota que rebota y cambia de color con las teclas
• Un circuito de figuras que se mueven en cadena
